Você é um especialista sênior em UX/UI, arquitetura de informação, design de portais institucionais governamentais e design systems. Sua tarefa é propor a **remodelação completa** de um site institucional existente, preservando sua identidade visual, mas modernizando totalmente sua estrutura, usabilidade e aparência.

O site atual é: https://www.comperve.ufrn.br/

Analise o site atual e use-o apenas como **referência de identidade visual e propósito institucional**, mas **não copie a estrutura atual**. O objetivo é propor um **portal moderno, organizado, acessível e escalável**.

CONTEXTO DO SITE
O site pertence à COMPERVE (Comissão Permanente do Vestibular da UFRN), responsável por processos seletivos como vestibulares, concursos, seleções e exames educacionais.

O novo site deve funcionar como um **portal público institucional**, com acesso fácil às informações de processos seletivos e com **um link claro para a Área do Candidato (sistema separado)**.

OBJETIVO
Gerar uma proposta completa de:
1. Estrutura do portal (arquitetura da informação)
2. Layout das principais páginas
3. Experiência do usuário
4. Paleta de cores
5. Tipografia
6. Componentes de interface
7. Diretrizes de design system

IMPLEMENTAÇÃO DA PROPOSTA

Para a demonstração do redesign, gere **um protótipo funcional simples composto por:**

- 1 arquivo HTML
- 1 arquivo CSS
- 1 arquivo JavaScript

Não utilizar frameworks de build, bundlers ou dependências complexas.

O objetivo é apenas um **protótipo visual funcional do portal**.

PROCESSO DE CRIAÇÃO

1. Crie **3 propostas diferentes de redesign**.

2. Cada proposta deve ter:
   - Conceito visual
   - Paleta de cores
   - Estrutura do portal
   - Layout da Home

3. Pelo menos:
   - **Uma proposta deve usar Bootstrap**
   - **Uma proposta deve usar Materialize**
   - **Uma proposta deve usar apenas CSS puro (sem framework)**

4. Após criar as três propostas:
   - Compare as três soluções considerando:
     - Clareza visual
     - Organização das informações
     - Usabilidade
     - Adequação a portal institucional
     - Facilidade de manutenção

5. Escolha **a melhor proposta**.

6. Gere ** o protótipo HTML + CSS + JS de cada solução**.

7. Além disso, gere um **arquivo adicional chamado `design-conventions.md`** contendo as **convenções utilizadas no design vencedor**.

8. Faça também um relatório comparativo em .md mostrando as diferenças de cada solução.

9. Cada solução deve ser alocada em pastas separadas e devem estar prontas para serem executadas quando executar o index.html.
